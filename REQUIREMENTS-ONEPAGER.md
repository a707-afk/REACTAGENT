# 企业知识库 RAG 问答（V1）需求一页纸

> 对应《AI应用开发项目建议与注意事项》中「项目 1」最小闭环前的需求定义，由项目负责人维护。

## 1. 给谁用

- **首要用户**：本人（学习与作品集演示）。
- **二期想象用户**：团队内需要查学习笔记、讲义与 RAG/Agent 相关 Markdown 的同事（不向公网开放，默认无多租户）。
- **个人 → 公司级演进（路线图）**：当前 V1 为 **单知识库、同步 API、Demo 级可用性与观测**；公司级能力（多租户、权限与审计、正式 SLA 数字、异步队列、混合检索与 Rerank）放入 **后续 V 阶段**，不在本里程碑展开。

## 2. 解决什么问题

- 在**已入库**的个人/学习文档中，用自然语言提问，获得**有依据**的答案，减少翻文件时间。
- 不在此版本中承诺替代专家判断或覆盖知识库外专业知识。

## 3. 输入 / 输出

| 类型 | 说明 |
|------|------|
| **输入** | 用户自然语言问题（中文为主）；可选后续扩展：指定 Top-K、是否只检索不生成。 |
| **输出** | **有检索证据且 Rerank 后门控通过时**：LLM 生成的回答正文 + **可追溯到 chunk 的引用列表**。**无结果、或重排后相关度低于阈值**：**不调 LLM**，返回默认 **「知识库中无相关内容」**（可配置）；引用为空；见 `refused`、`error_code`。 |

## 4. 无引用不强答（产品规则）

- **统一拒答话术（默认）**：**「知识库中无相关内容」**（可通过 `REFUSAL_NO_RESULTS` / `REFUSAL_GATE_FAIL` 覆盖；**拒答路径不调用 LLM**，便于审计）。
- **检索无结果**（`NO_RESULTS`）：返回上述固定模板。
- **K2（Rerank 后门控）+ 阈值（默认 0.6，须校准）**：先对向量宽召回结果做 **CrossEncoder 重排**，再在 **重排后的分数** 上判相关度：**最优重排分（Top-1）低于阈值** 则视为知识库无足够相关内容，走拒答（`SIMILARITY_GATE_FAIL` / `NO_SCORE` 等）。**不是**「向量召回结果里排名第 2 的 chunk」那一套语义。
- **分数语义**：向量 `score` 与 **重排分** 量纲不同；以 `python scripts/run_eval_retrieve.py` 输出的 **`summary.gate_score_*`**（及 `vector_raw_*` 对照）为准，再设定 **`RETRIEVAL_SIMILARITY_THRESHOLD`** 与 **`RETRIEVAL_SCORE_HIGHER_IS_BETTER`**。

## 5. 怎样算成功（V1）

- 本地可启动 API，对**已索引**文档中的事实性问题，多数情况下能返回**与文档一致**的答案。
- 每次回答附带**引用**；引用内容真实来自检索到的 chunk（后续 V 版本做自动校验）。
- 单次请求延迟与调用成本在可接受范围内；**本阶段为 Demo 级 SLA**（不以公司级 P99/可用性百分比作为验收）；**正式 SLA 数字放入 V 阶段**再与你方量化。

## 6. 什么情况必须拒答或降级

- **检索置信不足**：门控判定 Top 结果质量分低于阈值或与问题无关 → **无引用不强答**（见上）。
- **超出知识库范围**：与已入库文档无关的问题 → 明确说明无法依据当前知识库回答。
- **高风险误导**：制度/合规/医疗/财务等需人工审核类表述 → V1 仅作学习 Demo，回答中提示「仅供参考，重要事项以原文与人工确认为准」（产品化时再细化）。

## 7. 什么情况需要人工确认

- 用户要将答案用于**对外承诺、合同、审计、安全策略**等场景时，必须人工核对原始文档。
- 系统输出与原文存在歧义或引用片段截断导致模糊时，以**原文**为准。

## 8. 与 AI 能力边界的关系

- 不承诺 100% 正确；不承诺无知识库仍能准确回答专业问题。
- V1 定位为**辅助检索 + 辅助起草回答**，最终事实以源文档为准。
