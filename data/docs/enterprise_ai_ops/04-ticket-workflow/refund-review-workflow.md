# 退款申请审核流程（工单 / 财务）

> **域边界**：退款**材料清单、审批节点、财务复核字段**以本文为准。客户话术与「能否自动退款」见客服 FAQ；具体争议事件经过见 `07-cases/case-refund-human-review.md`。

## 适用范围

重复扣费、服务不可用、合同取消、套餐调整、计费争议等退款申请。**AI 不得自动批准退款金额。**

## 退款工单财务复核必填字段清单

进入财务审批前，工单必须包含以下 **财务复核必填字段**（缺一退回补料）：

| 字段 | 说明 |
|---|---|
| 客户编号 | CRM 唯一标识 |
| 合同编号 | 与 ERP 一致 |
| 付款记录 | 流水号、金额、日期 |
| 发票状态 | 已开/未开/红冲中 |
| 退款原因 | 客户陈述 + 内部分类 |
| 客户要求金额 | 币种与数值 |
| 影响说明 | 故障窗口或商务条款引用 |
| 相关沟通记录 | 工单与邮件摘要 |

缺少 **合同编号** 或 **付款记录** 时，不得进入财务审批。

## 流程步骤

1. 客服收集诉求并建单（标记 restricted）。
2. AI 生成摘要与材料清单（不批准金额）。
3. 客户成功确认商务影响。
4. 财务核对付款与发票。
5. 业务负责人审批。
6. 财务执行退款。
7. 客服通知客户。

## AI 可做 / 不可做

**可做**：检查材料完整性、生成摘要、提示缺失字段。  
**不可做**：承诺退款比例、到账时间、绕过财务审批。

## 与客户拒绝上传日志场景

若客户拒绝提供故障日志但仍要求立即退款，仍须走本流程；不得因客户施压跳过财务复核。相关历史案例见《案例：客户要求 AI 自动批准退款》。
