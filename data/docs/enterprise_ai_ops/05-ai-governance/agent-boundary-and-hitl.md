# Agent 行为边界与人工审核规范

## 基本原则

Agent 可以辅助理解、检索、总结、分类和建议，但不能在没有约束的情况下执行高风险业务动作。企业应用必须区分“建议”和“执行”。

## 适合 Agent 自动处理的任务

以下任务适合由 Agent 自动或半自动完成：

- 检索知识库。
- 总结工单内容。
- 识别问题类别。
- 生成回复草稿。
- 检查材料是否齐全。
- 生成下一步处理建议。
- 创建低风险待办。
- 提醒 SLA 风险。

## 必须人工确认的任务

以下任务必须由人工确认：

- 退款审批。
- 合同条款修改。
- 客户赔付。
- 账号权限提升。
- 数据删除。
- 生产变更。
- 安全事件关闭。
- 对外正式公告。
- 法律、财务、人事最终结论。

## 工作流控制

Agent 不应自行决定完整业务流程。推荐使用工作流引擎或 LangGraph 管理状态、分支和人工中断。

典型流程：

1. 接收用户问题。
2. Agent 判断意图和风险等级。
3. 调用 RAG 检索制度。
4. 生成建议。
5. 工作流判断是否需要人工审核。
6. 人工确认后执行工具。
7. 写入审计日志。

## 工具调用限制

工具应按风险分级：

- 只读工具：查询知识库、查询工单、查询公开状态。
- 低风险写工具：创建草稿、创建待办、添加备注。
- 高风险写工具：退款、删除、权限变更、生产变更。

Agent 默认只能调用只读工具和低风险写工具。高风险写工具必须经过人工审批。

## 审计要求

每次 Agent 决策应记录：

- 输入问题。
- 使用的工具。
- 工具参数。
- 检索证据。
- 风险判断。
- 人工审核结果。
- 最终执行动作。

这些记录用于事故追踪、模型优化和责任划分。
