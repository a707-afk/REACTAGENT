# AI Agent 记忆系统设计：分层记忆架构详解

## 为什么需要分层记忆？

最简单的 Agent 是"无状态"的——每次对话从零开始，结束就忘记一切。生产级 Agent 必须跨会话记住信息，但这带来三个核心挑战：

1. **记什么？** 不是所有信息都值得记住。用户纠正、项目规范、工具使用偏好等需要持久化；单次查询的临时上下文应该丢弃。
2. **怎么存？** 记忆不能无限增长——索引有上限，需要淘汰策略。
3. **怎么取？** 检索记忆不能占用太多上下文预算。全量加载不可行。

## 三层记忆架构

生产级 Agent 的记忆系统通常分三层，每层的持久性、信任度、审查需求不同：

### 第一层：指令记忆（Instruction Memory）

**定义**：人工策划的、分层的配置，按优先级注入系统上下文。

**层级结构**（优先级从低到高，高层可覆盖低层）：
- 组织级：公司编码规范、安全策略
- 用户级：个人偏好、常用工具
- 项目级：项目架构、依赖关系
- 本地级：当前会话的特殊指令

**特点**：
- 人类编写，稳定不变
- 注入时机：Bootstrap 阶段按优先级合并
- 不占用自动记忆的索引配额

### 第二层：自动记忆（Auto Memory）

**定义**：Agent 自主写入的持久知识，带有类型分类法和硬上限索引。

**写入协议（两步保存不变式）**：
1. 先写主题文件（原子性保证）
2. 再更新索引（包含文件路径和类型标签）

**类型分类法**：
- 用户纠正：用户明确指出的错误和修正
- 项目规则：从代码库中学习到的规范
- 工具偏好：某个工具在特定场景下的最佳参数
- 引用材料：用户提供的参考文档摘要

**索引上限**：约 50-200 条。超过上限时，新条目被静默截断——这是最常见的生产故障（Agent 看起来"忘了"新东西）。

### 第三层：会话提取（Session Extraction）

**定义**：后台 Agent 在会话结束时运行，自动从对话中提取关键信息写入自动记忆。

**工作流程**：
1. 主会话结束后触发
2. 后台 Agent 回顾整个对话
3. 提取：重要决策、用户偏好、新学到的规则
4. 写入自动记忆（遵循相同的两步保存不变式）

**并发安全**：互斥锁确保——如果主 Agent 在当轮已经写过记忆，提取器直接跳过。

## 审查与晋升机制

所有记忆层需要定期审查。审查 Agent 的工作：

1. 审计所有记忆层的一致性
2. 识别应从自动记忆晋升到指令记忆的条目（经常使用、高度可信）
3. 识别应降级或删除的条目（过期、冲突）
4. **关键**：永远不自主应用更改——只生成提议，需要用户明确批准

## 常见故障模式

1. **索引上限截断**：新记忆写完但索引已满，新条目不可见。Agent 反复"忘记"最近的纠正。
2. **写入不完整**：主题文件写成功但索引更新失败（崩溃/网络中断）。记忆存在但不可检索。
3. **提取冲突**：会话提取和自动记忆同时写同一主题文件，后写者覆盖前写者。
4. **上下文膨胀**：全量加载所有记忆到每轮上下文，token 成本线性增长。

## 面试回答要点

如果你的面试被问到"Agent 怎么跨会话记住信息"，关键要说清楚：
- 分层（不是只有一个记忆层）
- 写入协议（两步不变式保证数据安全）
- 索引上限和淘汰策略（不是无限增长）
- 审查晋升机制（不是永远留在自动记忆里）
