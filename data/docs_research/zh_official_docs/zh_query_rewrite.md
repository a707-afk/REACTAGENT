# Query Rewrite 策略：从口语到精确检索词

## 问题

用户输入："这个 Qdrant 过滤怎么搞啊，我要按部门查"。检索系统需要把它改写为"Qdrant payload filter 按 tenant_id 过滤用法"。

## 两层策略

### 启发式检测（零成本）
检测口语特征词（怎么、怎么搞、能不能、行不行、有没有、什么意思）→ 触发 LLM 改写。没有这些特征词的 query 直接跳过（不浪费 LLM token）。

### LLM 改写
System prompt："你是检索查询改写器。把用户口语改写为适合在技术文档库中检索的关键词。保留原意，使用技术术语。只返回改写后的 query。"

## 改写质量控制

改写后 query 必须保留原文的关键实体（产品名、技术名词）。改写失败的特征：改后 query 和原文关键词完全不同 → 退回原文。

## 多查询扩展

对复杂问题（"对比 A 和 B"），生成多个子查询而非单一 query：["A 的特性","B 的特性","A 和 B 对比"]。每个子查询独立检索，结果合并后 re-rank。

## 面试回答要点
"Query Rewrite 不是每次都调用 LLM——先用启发式特征判断是否需要改写，不浪费 token。复杂问题做多查询扩展而非单查询。"
