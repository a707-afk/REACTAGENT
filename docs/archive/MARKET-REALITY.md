## 一、真实 JD 拆解（30K 薪资段）

### JD-1：安续咨询 — AI大模型应用开发工程师（RAG方向） 15-30K×14薪
**核心职责**：
- 主导**企业级知识库构建**（不是 demo，是生产级）
- 向量数据库的架构设计、选型与**性能优化**
- 知识库数据**全流程处理**（获取→预处理→分块→向量化→索引→部署→监控）
- 优化索引结构（HNSW/IVF）及参数，平衡**召回率、延迟与资源开销**
- Docker/K8s 部署，Nginx 反向代理/负载均衡/HTTPS

**硬性要求**：
- 应届生勿扰，**无实际落地项目经验者勿扰**
- 深入理解向量索引算法（HNSW/IVF）原理，掌握参数调优
- 具备生产环境**性能优化能力**与**故障排查经验**

### JD-2：金仕达 — AI应用开发工程师（算法方向） 20-35K
**核心职责**：
- 基于大模型的AI应用，重点围绕 **AI Agent 智能体**构建业务系统
- Agent 的**工作流设计、工具调用集成、记忆管理**
- RAG 应用：向量数据库选型、文档处理管道、**召回策略优化**
- 性能优化：推理延迟、上下文处理效率、**稳定性问题**

**硬性要求**：
- **3年以上**后端或AI应用开发经验
- **必须有 Agent 或 RAG 实际项目经验**
- 理解 Agent 核心设计模式（**ReAct、Plan-and-Execute**）
- Python/TypeScript/Go 至少两种

### JD-3：半导体行业 — RAG/Agent工程师 18-35K×16薪
**核心职责**：
- 主导知识库**从0到1建设**
- 落地 **Agent 决策系统**：多步任务规划、故障根因分析
- 处理复杂文档（PDF表格、工程图纸、科学公式）

**硬性要求**：
- **硕士及以上**
- 复杂的 **Task Planning、Tool Use、Memory Management** 开发经验
- Docker/K8s + 模型量化与加速（vLLM, TensorRT-LLM）

## 二、残酷的现实分析

### 你的条件 vs 市场要求

| 维度 | 市场要求（30K） | 你的现状 | 差距 |
|------|----------------|---------|------|
| 学历 | 本科及以上（多数要求985/211或硕士） | 双非本科 | 学历是硬伤，但技术能力可以补 |
| 经验 | 3年以上后端/AI开发 | 1年 | 差距大，需要用项目深度弥补 |
| RAG落地 | 生产级，有性能优化和故障排查经验 | demo级 | 核心差距 |
| Agent | 必须有 ReAct/工具调用/记忆管理实战 | 直线流 Pipeline | 致命差距 |
| 工程化 | Docker/K8s/Nginx/监控 | 基本没有 | 需要补 |
| 语言 | Python + 至少一门（TS/Go/Java） | Python | 还行 |

### 诚实的结论
以你的条件（双非+1年），直接拿到 30K 的概率不高。但以下路径是可行的：

**路径A：先拿 15-20K 的 AI 应用开发岗积累实战经验**
- 15-25K 的岗位对经验要求相对宽松（1-3年）
- 先进去做半年到一年真实项目，再跳到 30K

**路径B：用项目深度弥补简历短板**
- 如果项目真的能做到下面"标杆项目"的水平
- 面试时能讲清楚每个决策的 why
- 有可量化的指标数据
- 有可能直接拿到 25-30K 的 offer（但竞争非常激烈）

**路径C（推荐）：A和B同时做**
- 一边深度打磨项目一边投简历
- 先面 15-20K 的岗位练面试、攒经验
- 拿到一个保底 offer 后，再冲击高薪岗位

## 三、能拿到 30K 的项目长什么样？（真实标杆）

### 标杆项目 1：Agentic RAG — 自适应检索 + 自纠正闭环

这是 2026 年面试中最受认可的项目类型。不是简单的"检索+生成"，而是：

```
用户问题
    ↓
┌─── Agent 决策循环 ───────────────────────────┐
│                                               │
│  ① Router（路由决策）                          │
│     LLM 判断：直接回答 / 检索知识库 / 联网搜索  │
│           ↓                                   │
│  ② Retriever（自适应检索）                     │
│     根据问题类型选索引：向量库 / BM25 / SQL / KG │
│           ↓                                   │
│  ③ Grader（文档评分）                          │
│     LLM 判断：召回的文档是否足够回答问题？       │
│     → 不够 → 改写 Query 重新检索（回到②）      │
│     → 够了 → 进入生成                         │
│           ↓                                   │
│  ④ Generator（受约束生成）                     │
│     基于检索证据生成回答 + 引用标注             │
│           ↓                                   │
│  ⑤ Hallucination Checker（幻觉检测）          │
│     检查回答是否有文档支撑                     │
│     → 有幻觉 → 带反馈重新生成（回到④）         │
│     → 通过 → 返回给用户                       │
│                                               │
│  终止条件：最大迭代次数 / 上下文充分 / 降级     │
└──────────────────────────────────────────────┘
```

**为什么这个项目能通过面试**：
- 它不是 Pipeline，是真正的 Agent — LLM 是决策者
- 有自纠正闭环 — 检索不够会重试，生成有幻觉会重来
- 有明确的终止条件和降级策略 — 说明你考虑了生产场景
- 每个节点都有量化指标 — 路由准确率、文档评分阈值、幻觉率

**面试中你要能讲的**：
- "Router 怎么决定的？" → 先用规则匹配简单 case（法条编号正则），再用 embedding 相似度，兜底 LLM 判断
- "Grader 的阈值怎么定？" → 在测试集上画 precision-recall 曲线，选 F1 最高的点
- "死循环怎么办？" → 最大迭代 3 次 + 连续重复检测 + 降级到直接回答并标记"不确定"
- "和普通 RAG 比提升多少？" → 复杂问题（多跳推理）准确率从 XX% 提升到 XX%

### 标杆项目 2：面试 Agent / 智能客服 Agent

来自真实面试复盘（拿到3个offer的人做的项目）：

```
系统功能：
  用户上传简历 → 系统解析 → RAG 从题库检索相关内容
  → 多 Agent 编排生成连续追问 → SSE 流式推送到前端

三个核心模块：
  1. RAG 知识库链路（题库/知识库的检索和排序）
  2. Agent 编排和状态机（多步对话的状态流转）
  3. 流式输出的稳定性（取消/超时/断线重连/可观测性）
```

**为什么能拿到 offer**：
- 有明确的业务场景（面试/客服），不是"通用问答"
- 涉及多 Agent 协作，不是单 Agent
- 涵盖了工程化难点：流式输出、状态管理、稳定性
- 面试时能用 STAR 法讲清楚

### 标杆项目 3：垂直领域知识库（金融/法律/医疗）

金融 RAG 的真实 JD 要求：
```
- 信息流自动打标
- 金融文档摘要生成
- 投资线索挖掘
- 智能投研助手
- 针对长文本、专业术语、数据稀疏的优化
- Milvus + ES 混合检索
- 领域知识库的持续迭代
```

**你可以做的法律版本**：
```
核心功能：
  1. 法律知识检索（法条/司法解释/案例/法律问答 四分库）
  2. 口语化→专业化的跨域检索（"欠钱不还多久过期" → "诉讼时效"）
  3. 多文档引用溯源 + 条款级别精确引用
  4. 基于 Agent 的多步法律推理（分析案情→检索相关法条→类案参考→给出建议）

技术深度：
  - 混合检索 BM25+Vector + RRF 融合（有 AB 数据）
  - 分库路由（规则→embedding→LLM 级联）
  - Agentic RAG 闭环（自适应检索 + 文档评分 + 幻觉检测）
  - ReAct Agent 工具路由（search_statute / search_case / search_interpretation）
  - 记忆分层压缩（长对话场景）
  - 全模块评估体系（检索/路由/生成/Agent 四层指标）
  - 三级缓存（精确/语义/LLM）
  - SSE 流式 + Docker 部署 + 可观测性
```

## 四、2026 面试真题实录（大厂考过的）

来源：腾讯云 2026年5月汇总，字节/快手/腾讯/Shopee 真实面试

### 必考题（每场都会问）
1. **RAG + Chat 算不算 Agent？**
   → 不算。Agent 的核心是自主决策，固定的"检索→生成"是 Pipeline

2. **ReAct、CoT、ToT 你用过哪个？实际效果怎么样？**
   → 不能只背概念，要讲实际用了哪个、为什么、效果提升了多少

3. **你的记忆机制怎么设计的？**
   → 长短期记忆怎么分、存在哪里、怎么检索、怎么压缩

4. **幻觉怎么检测和处理？**
   → 检测方法 + 处理策略（重新生成/标记不确定/转人工）

5. **工具调用失败了怎么办？**
   → 重试策略 + 降级策略 + 日志记录

### 架构设计题（区分度高）
6. **设计一个面向企业客户的智能客服 Agent**
   → 要讲：路由（常见问题/售后/工单）、RAG 检索、置信度校验、事实核查、人工复核、可观测性

7. **如果你的 RAG 系统流量突然涨 10 倍怎么办？**
   → 缓存层兜住重复查询、向量库增加副本、LLM 调用限流/排队、降级策略

8. **知识库更新了怎么同步到索引？**
   → 增量索引 vs 全量重建、缓存失效策略、版本管理

### 深度追问（能答好就拿高薪）
9. **RRF 的 k 参数怎么选？**
   → 网格搜索 + 在测试集上对比 Top-1/Top-5

10. **Embedding 模型怎么选？排行榜可信吗？**
    → 排行榜不可信，必须用自己的数据跑对比

11. **分块大小怎么定？**
    → 不同大小对 Top-K 命中率和 LLM 回答质量的影响，实验选最优

12. **你的系统成本怎么控制？**
    → token 消耗分析、缓存命中率、小模型处理简单问题

## 五、你的项目 vs 标杆项目的差距清单

| 能力项 | 标杆项目 | 你目前 | 优先级 |
|--------|---------|--------|--------|

## 六、最终建议

### 1. 调整薪资预期
双非+1年经验，第一份 AI 岗位目标建议定在 **15-22K**。这不是贬低你，是市场现实。进去之后做半年真实项目，跳到 30K 是完全可行的。

### 2. 项目重心调整
不需要两个项目，**一个深度足够的项目比两个浅的更有说服力**。把当前项目从 Pipeline 升级到 Agentic RAG 闭环，补上记忆/幻觉检测/评估体系/部署，就是一个有竞争力的项目。

### 3. 面试策略
- 先面 3-5 家你不太想去的公司，积累面试经验
- 每次面试后记录所有问题，找到自己的知识盲区
- 用面试反馈来指导项目优化的方向（而不是闭门造车）

### 4. 简历包装
- 不要写"个人项目"，写具体的功能和指标
- 强调"从 XX 优化到 XX"的定量结果
- 如果有博客/GitHub 活跃度，加分
